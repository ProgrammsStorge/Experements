## Эксперимент по созданию алгоритма предсказания текста

> Было 4 провальные попытки создать алгоритм, который мог бы отвечать хотя бы на тестовые данные.
> Это попытка №5, которая единственная смогла ответить на ряд тестовых запросов.

По своей сути это изменённая до неузнаваемости цепь Маркова с добавлением контекста.

**Описание работы алгоритма:**

1. Выборка опорных слов, к которым будет стремиться цепь.
2. Проход по словам:
   1. Сравнение того, насколько предыдущая часть до слова похожа на запрос.
   2. Выставление балла в зависимости от пункта 1 и от опорных слов.
   3. Добавление слова в запрос.
   4. Повтор до лимита `max_new_tokens`.

Алгоритм идеально воспроизводит входные данные и иногда генерирует новые.
Алгоритм требует много качественных входных данных для хорошей работы.

Входные данные должны быть разделены на контекстные блоки для правильной работы алгоритма. В контекстных блоках можно задать поведение: будет ли иметь память, сможет ли следовать инструкциям.

В алгоритме ещё много проблем, которые надо решить:

1. Плохая работа с научным текстом.
2. Сильная привязанность к диалогу.
3. Скорость работы.
4. Проблема гигантского открытого датасета.
5. Работа с шаблонами диалогов.
